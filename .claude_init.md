# Portfolio Project - AI Assistant Context

## Project Overview

**Name:** Developer Portfolio  
**Tech Stack:** Next.js 15 (App Router), TypeScript, Tailwind CSS  
**Package Manager:** pnpm  
**Target:** Vercel/Netlify deployment

## Architecture Principles

### 1. **Centralized Configuration**

- Site-wide configuration in `src/app/metadata.ts`
- Single source of truth for: name, URL, email, social links, SEO metadata
- All pages import from `siteConfig` and `defaultMetadata`

### 2. **Data Organization**

```
src/
├── app/
│   └── metadata.ts          # Site configuration (ALWAYS USE THIS)
├── data/                    # Static UI data
│   └── socialLinks.ts       # Social links with icons
├── components/
│   ├── ui/                  # Primitives (Button, Icon)
│   └── sections/            # Page sections
├── lib/
│   └── content.ts           # Markdown/JSON loaders
└── types/
    └── content.ts           # TypeScript definitions
```

### 3. **Component Pattern**

- **Server Components** (default): For static content, data fetching
- **Client Components** (`'use client'`): For interactivity (forms, buttons)
- Props-first: Pass data via props, not hardcoded values

### 4. **Type Safety**

- All types defined in `src/types/`
- Use TypeScript strict mode
- Interfaces over types for extensibility

## Critical Files

### `src/app/metadata.ts` (PRIMARY CONFIG)

```typescript
export const siteConfig = {
```

## Code Standards

### Imports

```typescript
// ✅ DO: Use absolute paths
import { siteConfig } from "@/app/metadata";
import { Button } from "@/components/ui/Button";

// ❌ DON'T: Relative paths
import { siteConfig } from "../../app/metadata";
```

### Metadata Pattern

```typescript
// Page metadata
import { siteConfig } from "../metadata";

export const metadata: Metadata = {
  title: `${PageName} - ${siteConfig.name}`,
  description: "...",
};
```

### Component Props

```typescript
// ✅ DO: Accept data as props
export function Footer({ socialLinks }: { socialLinks: SocialLink[] }) {
  return <div>{/* Use socialLinks prop */}</div>;
}

// ❌ DON'T: Import data directly in components
export function Footer() {
  const links = socialLinks; // Bad - breaks reusability
}
```

## Common Tasks

### Adding a New Page

1. Create `src/app/[page]/page.tsx`
2. Import `siteConfig` for metadata
3. Export metadata object with title pattern
4. Use Server Component by default

### Updating Site Information

1. Edit `src/app/metadata.ts` only
2. Changes propagate automatically
3. No other files need updating

### Type Errors

- **"ContactFromData"**: Should be `ContactFormData` (typo fixed)
- Run `pnpm tsc --noEmit` for type checking
- Run `pnpm build` to verify production build
